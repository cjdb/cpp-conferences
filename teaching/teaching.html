  • Try to understand the philosophy and history of C++ to explain why things are the way they are.

Style Guides

  • Any style guide that is unambiguously restrictive is a poor guide.
  • The C++ Core Guidelines recommend behaviour for the general-case, but don't outlaw anything, and especially not for arbitrary reasons.

Which C++ Standard?

  • C++98/C++03
  • C++11
  • C++14
  • C++17
  • C++next
  • C++98 is old, and should not be taught (except in very particular circumstances)
  • C++11 has been succeeded by both C++14 and C++17, and is starting to age
  • C++14 is a good starting point until Q4 2018
  • C++17 is the best choice, as of March, 2017
  • C++next is only a good choice when it has been all but ratified (like C++17 at present)

template <template <typename...> typename C, typename T>
auto sort(const std::vector<T>& add)
{
   auto c = C<T>{};
   for (const auto& i : add)
      c.emplace(ranges::find_if(c, [&i](const auto& x) {
         return x < i; }, i);

   return c;
}
